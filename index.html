Resumo das tags de cabeçalho h1 a h6

Função: Representam uma hierarquia de títulos, sendo o h1 o mais importante e seguindo até o h6

Como usar:

  • Navegação e Acessibilidade: Auxiliam na criação automática de tabelas de conteúdos e navegação em leitores de tela como o NVDA
  • É bom evitar múltiplos h1 na mesma página.
  • Estilizar é melhor num arquivo CSS e não no cabeçalho
  • As tags cabeçalho podem ser usadas para organizar seções e subseções de conteúdo.

O Elemento HTML "p" representa um parágrafo.

Corresponde a um conteúdo de fluxo, um conteúdo palpável. O que sua página quer apresentar.

São permitidos conteúdos de frases.

A tag de início é obrigatória. A tag de fechamento pode ser omitida se o elemento "p" for imediatamente seguido por um elemento de bloco ou se não houver mais conteúdo no elemento pai.

Para uso dinâmico, sua interface DOM é HTMLParagraphElement.

  • Referente à Acessibilidade:

    Dividir o conteúdo em parágrafos ajuda a tornar uma página mais acessível. Leitores de tela e outras tecnologias asssistivas oferecem atalhos para permitir que os usuários pulem para o próximo ou o parágrafo anterior, permitindo que naveguem pelo conteúdo, assim como o espaço em branco permite aos usuários visuais saltar entre o conteúdo. Não é recomendável usar elementos vazios, os leitores de tela podem anunciar a presença de parágrafo, mas não seu conteúdo. Ao precisar de espaço extra esse estilo deve ser colocado no arquivo CSS.

    • MARK:A TAG de texto com marcação

    O Elemento HTML mark representa um texto que está marcado ou destacado para referência ou propósito de anotação devido à relevância do trecho no contexto em que está inserido.

    Casos típicos de usos da Tag mark

    Quando usado em uma citação TQG q, ou citação em bloco TAG blockquote indica texto de interece especial que não está marcado na fonte original.

    É como usar uma caneta marca texto em um livro para destacar trechos de seu interesse.

    Mark não se confunde com strong o primeiro é usado para denotar conteúdo com algum grau de relevância, já o segundo é usado para destacar trechos de texto importantes.

      Referente à Acessibilidade

    • A presença do elemento mark não é anunciada pela maioria das tecnologias de leitura de tela em sua configuração padrão.
    • Uma maneira de termos esse anuncio é usando a configuração CSS content, junto com pseudo elementos ::before e ::after
    • mark::before {content: "[inicio do destaque]";} mark::after{content: "[fim do destaque]";
    • É importante ressaltar que o excesso de verbosidade afeta negativamente a compreenção dos textos na ferramenta de acessibilidade. Sendo assim é recomendada moderação no uso da tag.

    Small: O elemento de comentário lateral

    O Elemento HTML Small representa comentários laterais e texto em letras pequenas, como direitos autorais e textos legais.

    Por padrão ela renderiza o texto dentro dele em um tamanho de fonte menor, como de pequeno para muito pequeno.

    Pode ser alternativamente estilizado com Css, como em " Esta frase inteira em letras menores

      Referente à Acessibilidade

    • A tag small não afeta como os leitores de tela interpretam o conteúdo.
    • O texto dentro de uma tag small será lido da mesma forma que outro sem destaque especial.
