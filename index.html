O Elemento HTML "p" representa um parágrafo.

Corresponde a um conteúdo de fluxo, um conteúdo palpável. O que sua página quer apresentar.

São permitidos conteúdos de frases.

A tag de início é obrigatória. A tag de fechamento pode ser omitida se o elemento "p" for imediatamente seguido por um elemento de bloco ou se não houver mais conteúdo no elemento pai.

Para uso dinâmico, sua interface DOM é HTMLParagraphElement.

  • Referente à Acessibilidade:

    Dividir o conteúdo em parágrafos ajuda a tornar uma página mais acessível. Leitores de tela e outras tecnologias asssistivas oferecem atalhos para permitir que os usuários pulem para o próximo ou o parágrafo anterior, permitindo que naveguem pelo conteúdo, assim como o espaço em branco permite aos usuários visuais saltar entre o conteúdo. Não é recomendável usar elementos vazios, os leitores de tela podem anunciar a presença de parágrafo, mas não seu conteúdo. Ao precisar de espaço extra esse estilo deve ser colocado no arquivo CSS.

    • MARK:A TAG de texto com marcação

    O Elemento HTML mark representa um texto que está marcado ou destacado para referência ou propósito de anotação devido à relevância do trecho no contexto em que está inserido.

    Casos típicos de usos da Tag mark

    Quando usado em uma citação TQG q, ou citação em bloco TAG blockquote indica texto de interece especial que não está marcado na fonte original.

    É como usar uma caneta marca texto em um livro para destacar trechos de seu interesse.

    Mark não se confunde com strong o primeiro é usado para denotar conteúdo com algum grau de relevância, já o segundo é usado para destacar trechos de texto importantes.

      Referente à Acessibilidade

    • A presença do elemento mark não é anunciada pela maioria das tecnologias de leitura de tela em sua configuração padrão.
    • Uma maneira de termos esse anuncio é usando a configuração CSS content, junto com pseudo elementos ::before e ::after
    • mark::before {content: "[inicio do destaque]";} mark::after{content: "[fim do destaque]";
    • É importante ressaltar que o excesso de verbosidade afeta negativamente a compreenção dos textos na ferramenta de acessibilidade. Sendo assim é recomendada moderação no uso da tag.

    Small: O elemento de comentário lateral

    O Elemento HTML Small representa comentários laterais e texto em letras pequenas, como direitos autorais e textos legais.

    Por padrão ela renderiza o texto dentro dele em um tamanho de fonte menor, como de pequeno para muito pequeno.

    Pode ser alternativamente estilizado com Css, como em " Esta frase inteira em letras menores

      Referente à Acessibilidade

    • A tag small não afeta como os leitores de tela interpretam o conteúdo.
    • O texto dentro de uma tag small será lido da mesma forma que outro sem destaque especial.

    Tag i e tag u

    A tag i é um destaque do tipo itálico feito no trecho.

    A tag i já foi tipográfica, mas hoje é semântica. Ela representa uma parte do texto com uma difereça semantica na qual comumente se usa o itálico.

    Um exemplo é usar para termos técnicos ou idioma estrangeiro.

    A tag u é usada para um texto com notaçãonão articulada, como erros ortográficos internacionais ou para destacar uma palavra sem significado semântico adicional.

    Pode ser usada para erros intencionais ou gírias.

    O uso das tags i e u podem confundir os usuários que estão acostumados a serem direcionados a links com o destaque.

      Acessibilidade e NVDA

    • O NVDA não diferencia as tags i e u.
    • Ele faz a leitura normal como se elas não estivessem na página

    A TAG Strong

    Falamos do valor semantico da tag Strong quando comentaos sobre a TAG mark

    Apresenta em negrito o que estiver entre as tags de abertura e fechamento

    Sua interface DOM é HTMLElement, sendo que o Firefox implementa a interface HTMLSpanElement para este elemento.

    O elemento é utilizado em conteúdos que são de "grande importância", incluindo coisas de urgentes (como alertas).

      Acessibilidade NVDA

    • A documentação NVDA recomenda os elementos HTML adequados para marcar ênfases.
    • Isso pode ser posteriormente transformado de várias formas (alterações de estilo de fonte, alterações na inflexão da voz, etc.) por navegadores, leitores de tela e outros agentes de usuário.
    • Esses elementos devem ser usados em lugar dos elementos B e I.

    TAgs ol, ul e li

    Ambos os elementos ol e ul, representam uma lista de itens.

    Diferem porque, com o elemento ol, a ordem é significativa.

    Já o elemento HTML li é usado para representar um item que faz parte de uma lista. Este item deve estar contido em um elemento pai: uma lista ordenada , uma lista desordenada, ou um menu TAG:menu e representa uma única entidade dessa lista.

      Interfaces DOM:

    • UL: HTMLUListElement
    • LI: HTMLLIElement
    • OL: HTMLOListElement

    Blockquote: Elemento de citação de bloco.

    o Elemento indica que o texto incluído é uma longa citação.

    A URL para a fonte da citação pode ser dada usando o atributo cite, enquanto uma representação de texto da fonte pode ser dada usando a TAG cite elemento

    Utilize os elementos "Q" e "BLOCKQUOTE" para identificar as citações. A etiqueta "Q" deve ser usada dentro de um pequeno parágrafo para caracterizar uma fonte direta. A etiqueta "BLOCKQUOTE" deve ser usada para citações longas, que são recuadas das margens da página.

      Acessibilidade:

    • O uso correto das etiquetas de citação, Q e BLOCKQUOTE, pode quebrar a página em pedaços grandes de conteúdo, e ajuda os usuários a ler mais facilmente na tela
    • Alguns leitores de tela lerão o conteúdo identificado como citação com um tom de voz diferente, dando aos usuários pistas importantes sobre a estrutura do documento.
    • Atenção ao mal uso dos atributos de citação, por isso, confunde os usuários de leitores de tela como o NVDA
